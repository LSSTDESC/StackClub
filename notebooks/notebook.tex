    \section{Hello World}\label{hello-world}

In this tutorial we will: * Edit a notebook in the \texttt{jupyterlab}
environment * Commit and push those changes back to a development
branch, learning the DM Stack Club \texttt{git}/GitHub workflow.

    \subsection{Requirements}\label{requirements}

You need to be looking at this notebook from within a
\texttt{jupyterlab} environment. If you are, that means you have
successfully started a terminal via the Launcher, and cloned the
DMStackClub repo via

\begin{Shaded}
\begin{Highlighting}[]
\KeywordTok{git} \NormalTok{clone https://github.com/LSSTScienceCollaborations/DMStackClub.git}
\end{Highlighting}
\end{Shaded}

and are all set.

    \subsection{Editing this Notebook}\label{editing-this-notebook}

Add a single code or markdown cell below, with a suitable message for
the ages, including some sort of signature. This will make you famous.

    \begin{Verbatim}[commandchars=\\\{\}]

\end{Verbatim}


    \subsection{Contributing Your Work}\label{contributing-your-work}

Now that you have improved this Notebook, you need to ask for your work
to be included into the DM Stack Club library of community tutorials.
Here's the recommended workflow:

\begin{enumerate}
\def\labelenumi{\arabic{enumi}.}
\itemsep1pt\parskip0pt\parsep0pt
\item
  Check that this notebook runs without errors and clean out the outputs
\item
  Commit your changes to a development branch
\item
  Push your commits to a branch of the same name at GitHub.com
\item
  Submit a Pull Request to the \texttt{master} branch
\end{enumerate}

    \subsubsection{1. Checking the Notebook}\label{checking-the-notebook}

From the menu bar, do:

\begin{verbatim}
Kernel -> Restart and Run All Cells
\end{verbatim}

Then, if that worked OK, clean up the Notebook with

\begin{verbatim}
Kernel -> Restart and Clear All Outputs
\end{verbatim}

The Notebook is now ready to commit.

    \subsubsection{2. Committing Your
Changes}\label{committing-your-changes}

First, make sure you are on a development branch with a
\texttt{git status} in the terminal tab. If not, make one with something
like this:

\begin{Shaded}
\begin{Highlighting}[]
\KeywordTok{git} \NormalTok{checkout -b issue/11/hello-world-pjm}
\end{Highlighting}
\end{Shaded}

You may not be responding to an issue, so might want to name your branch
differently. Here's an example history:

\begin{Shaded}
